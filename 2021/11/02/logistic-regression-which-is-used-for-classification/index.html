Logistic regression: which is used for classification

Recently I am currently attending an online session on machine learning provided by Stanford University and taught by the great industry legend Andrew Ng. (only to my limited perspective). This is the content exclusively in week 3 so if you can’t catch up I might write the articles that introduce the chapters before this week in the future (actually most likely you’re never going to see them, my deep regrets).

Logistic regression is actually used to help us predict data samples in a classification problem, which requires us to label data samples as 0 or 1. For example, we need to classify a phone as an advertising spam call or just a regular call. Notice here there are only two types of possibilities available, being the call as spam or not spam. which is referred to as a binary classification problem. Now of course if we need to classify the call as an advertisement, a family call, or a delivery call it would not be binary then, but we’re not discussing this kind of problem at the moment, and for very simple and realistic reasons - I haven’t even learned them yet!

(Now now, you might ask, what does logistic regression have to do with regression? While even though logistic regression is used to solve classification problems, but the approaches taken during the process have been somehow involved with regression, but please don’t be confused by this concept. In fact, there is no need to remember this when implementating your solutions.) The question here is, the y value of our data samples can definitely be out of the range of 0 to 1. Thus we need to process the data samples in our data regression model to fit it in the range of 0 and 1. $$g(z) = \frac {1} {1+{e}^{-z}}$$ (Mathjax is such a good thing, am I right?) The function that is listed above is called the sigmoid function. Now actually the sigmoid function does not just include this function but includes some other functions such as the arctan function, but for the convenience of understanding, in machine learning usually, we just refer to this function only. The sigmoid function has a very unique ability that can map ANY number in the range from 0 to 1, which as you all know is the range of the possibility, indicating that now we can convert any real number to a value of possibility! Hooray!

Cost function

As you may all remember from the linear regression classes, you may recall that the general form of cost function is the equation below: $$\displaystyle J(\theta)=\frac{1}{2m}\sum_{i=1}^{m} \left(h_{\theta}(x^{(i)})-y^{(i)}\right)^2 $$ Yeah… We’re not gonna do that. If we apply this function to our data samples, we’ll end up having a curve of the cost function that is not actually convex - which means, there are multiple local minimums. We wouldn’t really favor that, since it would cause our little gradient descent algorithm to be stuck in a little hole. So we came with an exclusive cost function just for logistic regression: \begin{align*}& J(\theta) = \dfrac{1}{m} \sum_{i=1}^m \mathrm{Cost}(h_\theta(x^{(i)}),y^{(i)}) \newline & \mathrm{Cost}(h_\theta(x),y) = -\log(h_\theta(x)) \; & \text{if y = 1} \newline & \mathrm{Cost}(h_\theta(x),y) = -\log(1-h_\theta(x)) \; & \text{if y = 0}\end{align*} To be honest, I don’t know what the hell is this, so I decided to plot this function onto desmos. (In all the graphs, we are only concerned when x is smaller than 1 and bigger than 0.) This is the graph of log(x). Now the x here is actually a representation of $h_\theta(x)$, so you can see when $h_\theta(x)$ approaches zero, the cost will be close to negative infinity. If the $h_\theta(x)$ is 1 however, you can see the cost is actually zero. Why it is negative infinity? Please don’t ask me. In order to fix this, we need to add a negative sign in front of the log(x), which would bring it to -log(x). Now we have reflected the entire graph to the x-axis, so when $h_\theta(x)$ approaches zero, the cost would be positive infinity. Can you guess the y value from the graph? y equals 1. Because if $h_\theta(x)$ is 1, that proves our hypothesis right, which would bring the cost down to zero. (Yes!) But if you made a stupid mistake and predicted y as 0, well, you’re gonna have a cost even higher than admission fees to SCIE, which is infinity. Now, this is only the first scenario. The second scenario here is that y can be equated to 0, in this case, we need to implement a slightly different algorithm. Here is the scenario when y is 0. You can see the graph has been reflected to y = 0.5. When $h_\theta(x)$ is 0, the cost would be zero, and when $h_\theta(x)$ is 1, the cost would be infinity. Pretty easy to understand, huh? You can use this link to access the calculator if you wanna: https://www.desmos.com/calculator/4l0zc8t8lo

The vectorized version of the cost function is the following equation:

\begin{align*} & h = g(X\theta)\newline & J(\theta) = \frac{1}{m} \cdot \left(-y^{T}\log(h)-(1-y)^{T}\log(1-h)\right) \end{align*}

Multiclass Classification: One-vs-all

So we’re still discussing multiclass classification towards the end…

\begin{align*}& y \in \lbrace0, 1 … n\rbrace \newline& h_\theta^{(0)}(x) = P(y = 0 x ; \theta) \newline& h_\theta^{(1)}(x) = P(y = 1 x ; \theta) \newline& \cdots \newline& h_\theta^{(n)}(x) = P(y = n x ; \theta) \newline& \mathrm{prediction} = \max_i( h_\theta ^{(i)}(x) )\newline\end{align*} Now we are using a one-versus-all classfication algorithm. What we do is we seperate the examples into several binary classifiers(the number of the classfiers will be correspondent to the number of classes). We then run the sample into the classfiers and pick the classfier that gives the highest probability.
