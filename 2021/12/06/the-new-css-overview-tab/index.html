The new CSS Overview tab on Chrome(Edge)

It’s barely new anymore since it’s been out for more than one month and someone as blind as me just couldn’t notice the new tab. Oh well. [h2title]How to access this tab[/h2title] Literally, just press f12 on your browser and find your tab right away. I think my chrome is not updated so I’m gonna use edge here. [h2title]How to use it[/h2title] Let’s say you’re a talented web designer but with absolute zero art taste. You decide to design a little functional website for your entire school to use. And it looks a bit like this…: I know what you guys are thinking. This is a terrible color combination! But lucky for you, with your IMPECCABLE color palette, you can’t even see your own mistakes. Now the CSS Overview can help you with this problem!(

This tab consists of five parts:

Overview summary: This provides information for the total amount of elements, selectors, and style rules on your page. Not sure why you absolutely need those statistics though…

Colors: This tab shows you detailed information on all the colors on your website, including background colors and text colors. This tab also suggests improvements on the contrasting background color and text color, so you can adjust your palette to make your website more human-friendly. When you click on the colors, the browser will show you a list of elements that all use this color. Also, I investigated something I found pretty intriguing. Notice here we add a completely useless cyan box. If we apply this box with a “display: none;”, the CSS overview would not take this element into the consideration of colors. However, if we apply an “opacity: 0;” or “visibility: none;” onto the box, the CSS overview actually considers the box as a color, even though it’s not shown at all. Please notice that the website still looks like this: I know probably no one really cares about this cute little interaction, but this still demonstrates some difference between the realization of different methods in CSS, though they all achieve the same effects.

Font info: This is kind of similar to the colors tab, as it lists all the fonts and their corresponding attributes on the page. You can also click on the occurrences and investigate which element is using the font.

Unused declarations: This tab lists all the style rules that actually have no use at all or contradict with other style rules on an element. An independent web designer that starts their website from scratch may not notice this as much as an error, but in integrated platforms (like WordPress) these mistakes may occur. Again, you can click on the occurrences and find the location of your errors…

Media queries: Lastly, the overview also offers a tab that lists all media queries descending from the highest occurrences to the least. Most of them may be from other scripts, such as bootstraps, or other stylesheets.

[h2title]Hope for a better future[/h2title] This function is actually only in preview, so it may not be sufficient enough to meet the demands of some front-end developers. For example, a function I would love is to change the color of all the elements with the same color into a different color, which can instantly change the style of the entire web page. Some recommended color schemes can also be listed in this tab so web designers with zero artistic instincts (such as me, especially me) can also design a website that’s suitable for humans to visit, not aliens.
